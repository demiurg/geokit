Tables

GeoKit offers users the ability to submit and store tabular data formatted as comma separated values. This functionality gives users the opportunity to visualize data from sources other than those provided by GeoKit's raster data service. This is a great way for researchers to visualize data for the web without programming and various IT concerns.

Submitting Tabular Data

This entry about tabular data should provide a brief description of how the data should be formatted, and provide a visualization of that formatting.

  1. Name: Tables will be selected by name during variable creation, so this should be something recognizable
  2. Description: GeoKit sites with many tables may find this optional field useful for distinguishing tables
  3. CSV File: Click the "Choose File" button to select the CSV file to be uploaded to GeoKit

Layers

GeoKit layers are collections of spatial coordinates that define the features that users would like to visualize data for. Layers can be created from shapefiles and GeoKit's vector catalog, which is an interface for interacting with various spatial databases, giving users many options for defining the bounds of their visualizations.

Creating Layers from Shapefiles

From the Wikipedia entry for Shapefiles:

The shapefile format is a popular geospatial vector data format for geographic information system (GIS) software. It is developed and regulated by Esri as a (mostly) open specification for data interoperability among Esri and other GIS software products. The shapefile format can spatially describe vector features: points, lines, and polygons, representing, for example, water wells, rivers, and lakes. Each item usually has attributes that describe it, such as name or temperature.

Users with GIS software experience can work with existing features by uploading their shapefiles to their GeoKit accounts.

  1. Name: Layers will be selected by name during variable creation, so this should be something recognizable
  2. Description: GeoKit sites with many layers may find this optional field useful for distinguishing layers
  3. Vector file: Click the "Choose File" button to select the shapefile to be uploaded to GeoKit

Creating Layers from the Vector Catalog

GeoKit offers the Vector Catalog interface for interacting with various sources of spatial data. The vector catalog renders this spatial data onto an interactive map. GeoKit users can then select the features to be included in their layers.

  1. Name: Layers will be selected by name during variable creation, so this should be something recognizable
  2. Data Source: Defines the spatial configuration of the features
  3. Map: Select features to be included in the layer by pointing and clicking on individual features, or by using the geometrical selection tools provided in the map toolbar

Variables

GeoKit variables are the marriage of spatial data with temporal observational data. GeoKit combines the spatial features defined by users in the layers they've created with tabular data provided by users or raster data provided by a GeoKit service that serves as an interface to vast NASA geospatial observational databases.

Spatial Configuration

GeoKit offers the Vector Catalog interface for interacting with various sources of spatial data. The vector catalog renders this spatial data onto an interactive map. GeoKit users can then select the features to be included in their layers.

Steps:

  1. Select a layer from the dropdown

Upon selection, the layer will be rendered on the map so that users can visually confirm that they've selected the correct layer.

Data Configuration

Users are prompted to select either a tabular or raster data source. Users who will be visualizing data that they've uploaded to GeoKit should opt to use user-submitted data, while those who wish to use data from NASA observational databases should opt for using GeoKit data.

Data Configuration - Tabular

Tabular data must be joined to spatial data using a common data attribute. A user would have a table with one of the columns being county identifier, such as name of FIPS id. The spatial layer would also have an identifier for each spatial unit. User would select pick the column name from the table to match the spatial layer identifier to match.

Data Configuration - Raster

Raster data are available from many sources, which are listed in a table that describes the data and their temporal range of availability.

Steps:

  1. Select the data type from the table
  2. Configure the temporal range for the data
  3. Name the data
